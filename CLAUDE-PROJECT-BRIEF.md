## Bilingual system (DE/EN)

- Root element: `<html class="de">` or `<html class="en">`
- Elements with `.only-de` / `.only-en` are shown/hidden via CSS
- `data-en="..."` attribute on elements: main.js swaps `textContent` on language toggle
- **Rule:** ALL user-facing text must have both DE and EN versions in JSON and HTML
- **Rule:** NO Arabic characters in any code file (JS/HTML/CSS/YAML/TOML/JSON)

### Project JSON format
Each project lives at `content/{section}/{slug}.json`:

```json
{
  "order": 10,
  "title_de": "...",
  "title_en": "...",
  "summary_de": "...",
  "summary_en": "...",
  "cover": "assets/uploads/filename.jpg",
  "video": "https://www.instagram.com/reel/CODE/embed/",
  "aspect_ratio": "9:16",
  "media_width": "medium",
  "chips": ["Tool A", "Tool B", "Tool C"],
  "body_de": "Long text...",
  "body_en": "Long text...",
  "gallery": [
    { "image": "assets/uploads/img.jpg", "aspect_ratio": "auto", "size": "1" }
  ],
  "link_url": "https://...",
  "link_label": "Label ↗",
  "tracks": [
    {
      "id": "Track 01 · Label",
      "title_de": "...", "title_en": "...",
      "text_de": "...", "text_en": "...",
      "chips": ["Tool"]
    }
  ]
}
```

**Field notes:**
- `order`: lower = appears first in grid
- `video`: use Instagram reel embed URL (`https://www.instagram.com/reel/CODE/embed/`) or YouTube embed URL. Do NOT use full `<iframe>` HTML.
- `aspect_ratio`: `"9:16"` for vertical reels, `"16:9"` default (no override needed), `"1:1"`, `"4:3"`, `"3:4"`
- `media_width`: `"small"` (380px), `"medium"` (560px), `"large"` (760px), or omit for full width
- `gallery.size`: `"1"` = normal, `"2"` = spans 2 columns
- `chips`: always an array of individual strings — NOT one long joined string
- `tracks`: production pipeline steps shown in the modal — optional but recommended

### Adding a new project (workflow)
1. Copy images to `assets/uploads/`
2. Create `content/{section}/{slug}.json` with full DE+EN content
3. Add `"{slug}"` to the correct section's `items` array in `content/sections.json`
4. Run `node build-work.js` (Claude can do this via Bash tool) to regenerate `data/work.json`
5. Verify in local preview at `http://localhost:4444/work.html`
6. Push via GitHub Desktop

## Logo & branding

- Logo image: `assets/uploads/my-logo.png` — hardcoded directly in all HTML `<header>` elements as `<img class="brand-img">`
- Logo height: controlled by CSS variable `--logo-h`, currently `50px` (confirmed final value by Saleem via DevTools check) at style.css line ~22
- **Important history:** the original `my-logo.png` was a 4500×4500 canvas with huge transparent padding around the real artwork (`getbbox()` showed real content was only 3957×1743 inside it). This made any `--logo-h` setting render the logo small/soft while still eating header height. Fixed by cropping the PNG to its bounding box + 40px padding (now 4037×1823). If the logo ever looks "blurry" or the header looks "too tall" again after a future logo swap, re-check for this same padding issue before touching CSS.
- Header height: `.site-head .wrap{height:80px}` in style.css — this is the correct/normal value; do not bump it to fix logo sizing, fix the source PNG instead.
- `content/site/branding.json` — CMS-editable branding (can override logo/favicon at runtime)
- Favicon: `assets/favicon.svg` (default). Can be overridden via branding.json

## Known issues & decisions

| Topic | Decision |
|---|---|
| Instagram embeds | Use `https://www.instagram.com/reel/CODE/embed/` as `video` field value. YouTube player was rejected for looking "messed up". |
| Reveal animation bug (fixed) | Cards injected by work.js were invisible because IntersectionObserver ran before DOM was ready. Fixed: `window.__observeReveal()` called after `mount.innerHTML = html` in work.js |
| Gallery field format | Gallery items MUST be objects `{"image":"...","aspect_ratio":"...","size":"..."}`. Old string format `["path"]` crashes Decap CMS. |
| Credits conservation | `editorial_workflow: true` in admin/config.yml — CMS Save = draft branch (no deploy). Only Publish = deploy. |
| Section cover cards | Removed. Each section now shows project cards directly in a clean grid. |
