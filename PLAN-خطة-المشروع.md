## أين وصلنا (الحالة الحالية)

الموقع **منشور وشغّال** على رابط Netlify مؤقت مخفي (`اسم-عشوائي.netlify.app`). دومين saleemrishah.com **لسا ما ارتبط** (مقصود — لحد ما يجهز كل شي).

ما تم إنجازه بالكامل:
- صفحة Work أُعيد بناؤها: 4 أقسام (AI / Motion / Brand / Football). جوليا صارت عمل ضمن AI، مو قسم متفرّد.
- كل عمل = كرت قابل للضغط يفتح نافذة فيها: صور + فيديو embed + وصف + chips + tracks (لجوليا).
- باند جوليا المتصدّر انشال من الصفحة الرئيسية.
- لوحة تحكم Decap CMS شغّالة (واجهة إنكليزية): تتحكم بالأعمال + الصفحة الرئيسية + اللوغو + الأيقونة، بدون كود.
- الموقع مرفوع على GitHub + Netlify، ولوحة التحكم مفعّلة (Identity + Git Gateway).

## طريقة العمل (مهمة — احفظها)

- **كل تعديل** (سواء من اللوحة بـ Save، أو من الكود بـ Push) = "build" واحد على Netlify = استهلاك كريديت بسيط.
- **ما يستهلك كريديت:** تصفّح اللوحة، الكتابة قبل الحفظ، زوّار الموقع، فيديوهات يوتيوب المضمّنة.
- **القاعدة الذهبية:** نجمّع التعديلات وننفّذها دفعة واحدة، ونرفعها برفعة واحدة (build واحد) — توفيراً للكريديت والوقت.
- خطة Netlify المجانية (300 كريديت/شهر) تكفي الموقع بسهولة.

**خطوات رفع أي تعديل من GitHub Desktop:**
1. عمود Changes يبيّن الملفات المتغيّرة.
2. اكتب وصف بخانة Summary.
3. اضغط Commit to main.
4. اضغط Push origin.
5. انتظر دقيقة-دقيقتين (Netlify يبني تلقائياً).
6. حدّث الصفحة بـ Ctrl+Shift+R.

## التعديلات المعلّقة (دفعة واحدة — جاهزة للتنفيذ)

### تعديلات تصميم
1. **مقدمة صفحة Work:** ترجع بنمط الهوم (إيبرو "What I do" + عنوان كبير h2 بنفس الخط والحجم). بصياغة مختلفة عن قسم AI حتى ما يصير تكرار.
2. **قالب الصورة الشخصية:** يحتوي أي صورة بنفس الحجم (لو أكبر)، شيل الغريدينت، ودعم PNG بدون خلفية (ما تطلع خلفية).
3. **صور وفيديوهات المشاريع:** تخطيط ذكي تلقائي (شبكة متوازنة) — الموقع يرتّب الكتل لحاله، بدون ما تحسب أحجام.
4. **توحيد الخطوط:** التأكد إنّو عناوين/نصوص المشاريع تستخدم خطوط الموقع الافتراضية (تجانس كامل).

### تعديل قانوني
5. **Datenschutz:** تعبئة اسم مزود الاستضافة الفعلي (Netlify) بالقسم 3، وحذف صندوق التنبيه الأصفر (.notice).

## خطة المحتوى (كيف نحكي عن كل مشروع باحترافية)

لكل عمل، البنية الاحترافية المقترحة:
- **العنوان:** اسم المشروع + نوعه (مثلاً "Julia Kallas — KI-Persona").
- **الوصف القصير (على الكرت):** جملة واحدة تشدّ الانتباه، تقول شو هو وليش مميّز.
- **الوصف الطويل (داخل النافذة):**
  - *التحدّي/الهدف:* شو كانت المشكلة أو الفكرة.
  - *الحل/المنهجية:* كيف اشتغلت عليه، أي خطوات.
  - *الأدوات:* تُعرض كـ chips (وسوم).
  - *النتيجة:* أرقام أو تأثير ملموس.
- **القاعدة:** نُظهر القدرة (capability) مو بس الناتج. الـ HR الألماني يهمّه "كيف فكّرت" مو بس "شو طلع".

تقرير المحتوى الكامل والمفصّل سيُكتب بعد ما نخلّص التصميم.

## الأمان (بعد الجهوزية)

إجراءات سنطبّقها معاً قبل ربط الدومين:
1. **HTTPS تلقائي:** Netlify يفعّله لحاله عند ربط الدومين (شهادة Let's Encrypt مجانية).
2. **2FA على حسابك (GitHub + Netlify):** تفعيل المصادقة الثنائية على الحسابين — أهم خطوة حماية.
3. **Identity على "Invite only":** (مفعّل أصلاً) — ما حدا يقدر يسجّل دخول للوحة غيرك.
4. **Security headers:** عبر ملف netlify.toml — حماية من clickjacking وحقن المحتوى.
5. **سجل CAA بالـ DNS:** يمنع أي جهة غير Netlify من إصدار شهادات لدومينك.

> الموقع ستاتيكي (HTML/CSS/JS فقط) — يعني سطح الهجوم صغير جداً أصلاً. ما في قاعدة بيانات ولا سيرفر backend يُخترق.
